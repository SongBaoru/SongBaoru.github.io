虚拟头节点

链表删除元素的操作有两种操作方式：

  1. 直接使用原链表执行删除操作
  2. 设置一个虚拟头节点再执行删除操作

对于第一种操作方式，删除头节点和删除其它节点的操作是不一样的，因为链表中其它节点是通过前一个节点来删除的，而头节点没有前一个节点，在这种操作方式下，头节点的删除是通过将头节点向后移实现的。

对于第二种操作方式，为链表设置一个虚拟头节点后，原链表的所有节点都可以按照统一的方式删除。

哈希法

如果在做算法题目时，遇到需要判断元素是否出现过，或统计出现次数的场景，则应该想到使用哈希法。

Java中哈希法使用到的数据结构包含以下几种：

  • 数组：适合处理元素数量有限的场景。
  • HashSet：只适合判断元素是否出现过的场景。
  • HashMap：可以处理所有哈希法的适用场景。如果数组和HashSet能够处理，则往往使用数组和HashSet更加方便和高效，此时应优先使用数组和HashSet。
  • 数组、HashSet、HashMap的“扩展类”，如List、TreeSet、TreeMap等。

KMP算法

KMP算法主要应用在字符串匹配的场景中，核心思想是当出现字符串不匹配的情况时，可以利用模式串的文本信息，避免再从头匹配。在KMP算法中，该文本信息记录在next数组中。next数组是一个前缀表（prefix table），或者是前缀表的某种变形（如前缀表的元素统一减一后的前缀表、前缀表元素整体后移一位并在0位补-1），前缀表记录的是最长相同前后缀的长度的信息。根据前缀表，可以得知当模式串与文本串不匹配时，应该从模式串的哪个位置开始继续遍历匹配，既不会遍历到不可能存在匹配子串的部分，又不会漏掉可能存在匹配子串的部分。

如何计算前缀表（最长相同前后缀是长度表）

前缀表就是从当前下标往前的所有模式串字符的最长相同前后缀的长度。

计算方法是：定义两个指针i和j，i用于遍历next数组，j用于指向模式串的下标，j需要更新到满足条件，模式串(i - j, i]范围内的子串，与模式串[0, j)范围内的子串匹配，当满足这一条件时，填充next[i]=j，即从i位置向前数j个元素是i位置的最长相同前后缀。为满足这一条件，会在前后缀字符不匹配时，使用next数组已经得到的部分结果更新j的值。

以模式串acbacf为例，计算前缀表的结果：

前缀子串 前缀 后缀 匹配的子串 i j 最长相同前后缀的长度
ac 0
acb 0
acba axxx xxxa a 3 1 1
acbac acxxx xxxac ac 4 2 2
acbacf 0

得出模式串acbacf的前缀表为[0, 0, 0, 1, 2, 0]。

如何使用前缀表

在while循环中判断，模式串的下标j处和文本串的下标i处字符是否匹配，如果不匹配，则更新j的值，使j=next[j-1]，当然，循环条件中还要有j是否大于0，如果j小于等于0，j-1就是负数，下标越界，所以j需要大于0。

时间复杂度分析

设m为模式串的长度，n为文本串的长度，KMP算法中生成模式串的时间复杂度为O(m)，进行字符串匹配的时间复杂度是O(n)，所以总的时间复杂度是O(n+m)；而暴力解法的时间复杂度是O(n×m)，所以KMP大大加快了字符串匹配的速度。

每日温度（单调栈）

用法示例（力扣739. 每日温度：根据每日温度数组，得出在数组的每个位置，要想观测到后续出现更高的温度，需要等待的最小天数）：

这道题可以使用单调栈解决，如何判断能否使用单调栈？通常是一维数组，要寻找任意一个元素的右边或者左边比自己大或者小的元素的位置，此时就要想到可以使用单调栈。单调栈的时间复杂度为O(n)。

这道题其实就是，要寻找任意一个元素的右边比当前元素大的元素，所以应该想到使用单调栈。

单调栈的本质是空间换时间，使用了一个栈来记录遍历过的元素信息。

使用单调栈前需要明确的两个点：

  1. 单调栈里存放的是什么：单调栈里存放的是元素的下标。
  2. 单调栈是递增的还是递减的：
    1. 如果求的是一个元素左边或右边第一个更大的元素，则单调栈是递增的。
    2. 如果求的是一个元素左边或右边第一个更小的元素，则单调栈是递减的。

这道题解法的具体操作是，如果当前元素大于栈顶元素，就把栈顶删除，加入当前元素；如果当前元素小于等于栈顶元素，则加入当前元素，可见栈顶一定是当前元素的下标。

单调栈解法

与前两种解法不同，这道题的单调栈解法是按照行的方向计算雨水面积，因为单调栈存储的是元素的下标，所以能通过单调栈确定左右两侧柱子的距离差和高度差，从而求出面积。其实就是栈顶和栈顶的下一个元素以及要入栈的元素，三个元素来接水！

在入栈过程中有三种情况：

  1. 入栈的元素小于栈顶元素，则将元素的索引入栈。
  2. 入栈的元素等于栈顶元素，则将栈顶出栈，将当前元素的索引入栈。之所以要将栈顶出栈，是因为单调栈内的元素的索引是作为左边界出现的，两个高度相等的相邻的柱子，左边一个是不能存放雨水的。
  3. 入栈的元素大于栈顶元素，则将栈顶元素弹出，并计算由栈顶和栈顶的下一个元素以及要入栈的元素，所组成的行方向上的面积。

二叉树的种类：

贪心法

贪心的本质选择每一阶段的局部最优，从而实现全局最优。难点是如何通过局部最优推出全局最优，最常用的办法是举反例，如果想不到反例，就可以试一试贪心算法，严格的使用数学归纳法或反证法的数学证明不常用。

使用贪心算法解题一般分为如下四步：

  1. 将问题分解出若干子问题。
  2. 找出适合的贪心策略。
  3. 求解每一个子问题的局部最优解法。
  4. 将局部最优堆叠成全局最优。

回溯法

回溯法也称回溯搜索法，是一种搜索方式。递归法的递归过程中就有回溯的过程。回溯法解决的问题都可以抽象为树形结构，因为回溯法解决的问题都是在集合中递归查找子集。集合的大小就构成了树的宽度，递归的深度构成了树的深度。

回溯法的本质是穷举（暴力解法），并不高效，当然也可以增加一些剪枝操作，提高效率。之所以不高效还使用，原因是部分问题只能通过穷举再加剪枝解决，没有更高效的解法。

这些只能通过穷举解决的问题包括：

  1. 组合问题：在n个数中找出所有符合规则的k个数的组合。
  2. 子集问题（组合问题的问法变种，跟组合问题的解法是一样的，只是没有了任何长度和总和的限制，所有组合都能往结果集里面添加）：在n个数中找出所有符合条件的子集。
  3. 子序列问题：：在n个数中找出所有符合条件的子序列，子序列和子集的不同是，子集是无序的，子序列是有序的。
  4. 排列问题：在n个数中找出所有符合条件的排列方式。
  5. 分割问题：在一个字符串中找出所有符合规则的分割方式。
  6. 棋盘问题：N皇后，数独问题。

组合是不强调元素顺序的，而排列强调元素顺序。

回溯法“三部曲”（回溯法模板）：

  1. 确定回溯函数的返回值和参数。

    回溯函数的函数返回值一般是void。

    因为回溯函数的参数不容易一次性确定下来，所以一般是先写逻辑，然后需要什么参数就增加什么参数。

  2. 确定回溯函数的终止条件。

    由于回溯法解决的问题都可以抽象为树形结构，所以一种情况是，搜索到叶子节点也就找是到了满足条件的一个答案，此时应把答案存入结果集，并结束本层递归。

  3. 确定回溯搜索的遍历过程。

    输入的集合的大小构成了树的宽度，递归的深度构成了树的深度。

限定长度的组合（组合问题）

用法示例（力扣77. 组合：返回[1, n]范围内的所有长度为k的组合）：

如果使用暴力解法，那么for循环的层数和k一样，k可能很大，导致循环嵌套层数过多，而回溯法使用递归解决了循环嵌套层数过多的问题。

要对回溯法解法进行分析，可以将问题抽象为树形结构（N叉树），以便于理解。对于每个节点的子节点，从左往右，可以选择的元素的个数依次递减（因为左侧的元素已经在左边的兄弟节点遍历过了，只需要继续遍历右侧的元素）。

按照回溯法“三部曲”分析如下：

  1. 确定回溯函数的返回值和参数。

    需要一个变量存放查找结果（设为result），需要一个变量存放已经遍历的部分（设为path），需要两个参数传递输入的n，k，另外还需要一个参数存放遍历的起始下标（设为startIndex），可以理解为N叉树中当前节点在其所有兄弟节点中的位置下标。

    返回值是void。

  2. 确定回溯函数的终止条件。

    如果path的长度大小等于k，说明找到了一个长度为k的组合。此时使用将path存入result，并结束当前的递归分支。

  3. 确定回溯搜索的遍历过程。

    遍历从startIndex开始，先使用path保存遍历的节点，执行完新的回溯分支后，删除path在进入回溯分支前保存的节点（回溯）。回溯的目的是保证path中不会添加兄弟节点，因为兄弟节点已经在新的回溯分支中遍历过了，如果不删除，会出现三个问题：

    1. 添加到result内的组合重复

      兄弟节点已经在新的回溯分支中遍历过了，如果不删除，后续的组合会出现和之前元素一样的组合，如果这种组合恰好满足条件，就会被重复添加到result中。

    2. 部分组合不能搜索到

      首先对于包含左侧元素的组合就不能搜索到，因为如果不删除，左侧的元素一定会保留在path中。

    3. N叉树不能正确分支

      因为遍历中的每次取值都是一个分支，如果不删除，则进入分支时不能保证只有一个节点。

    这道题还可以对遍历进行剪枝，分析得到能够剪枝的情况是，如果剩余的元素个数（n-i）小于还需要的元素个数（k-path的长度），就没必要继续遍历了。

要保证结果集中不含有重复组合，一种解决办法是找出所有组合之后，使用set或map去重，但是这样做效率低，所以考虑在搜索的过程中完成对组合的去重。在搜索过程中去重，相当于在同一数层的元素不能重复选取。去重操作在树形结构上分两种维度，一个维度是同一树枝，另一个维度是同一数层。判断是否使用树枝维度去重的依据是，数值相同的元素在一个组合内是否可以重复，如果不能重复，则需要使用树枝维度去重；判断是否使用数层去重的依据是，组合是否可以重复，如果不能重复，则需要使用树层维度去重，如果只要使用的不是同一个元素那么组合就可以重复，则不需要去重。当然前提都是给定集合内有重复元素，如果没有重复元素，就不需要去重了。需要强调的一点是，去重一定要对元素进行排序，这样才能通过相邻的节点来判断元素是否重复使用了。

因为本题中，数值相同的元素在一个组合内可以重复（出现多次），所以不需要对树枝维度去重；两个组合不能重复，所以需要使用对数层的维度去重。

    本题需要在上一题的基础上，增加一个bool类型的数组（设为used），用来记录同一树枝或树层上的元素是否使用过，used[i - 1]为true表示元素在同一树枝使用过，used[i-1]为false表示元素在同一树层使用过。还可以使用startIndex去重，下标大于startIndex表示元素在同一树层使用过，但使用used数组去重具有普适性，也更容易理解。

    返回值是void。

  2. 确定回溯函数的终止条件。

    这道题的终止条件和上一题相同。

  3. 确定回溯搜索的遍历过程。

    这道题的回溯搜索的遍历过程和上一题主要的区别有两个，一个是不能对元素重复使用，所以进入下一层回溯函数迭代时，需要对索引加1；另一个是多了去重。

动态规划法

动态规划（Dynamic Programming，简称DP）是解决具有重叠子问题的问题的最有效的解决方法。

动态规划中的每一个状态一定是由上一个状态推导出来的。这一点就区别于贪心法，贪心法没有状态推导，而是直接选择的局部最优。

动态规划法的“五部曲”（动态规划法的解题步骤）：

  1. 确定dp数组及其下标的含义。
  2. 确定递推公式。题目的难易程度很大程度上与递推公式的难易程度正相关。
  3. 确定遍历顺序。如果dp后面元素的取值依赖于dp前面元素的取值，则遍历顺序是从前往后，反之，遍历顺序是从后向往前。
  4. 初始化dp数组。递推公式决定了如何初始化dp数组，所以确定递推公式要在初始化dp数组之前。
  5. 举例推导dp数组。做动态规划的题目，写代码之前一定要把dp的递推过程先在脑海中模拟一遍，做到心中有数，确定最后推导出的是想要的结果，然后才编写代码。如果代码没通过，就打印结果，查看和自己预先推导的结果是否一样，如果不一样，就是代码的实现有问题。

0-1背包问题的特点是每种物品只有一个。0-1背包问题可以使用回溯法进行暴力求解，时间复杂度是2的n次方，是指数级别的，使用动态规划能够降低0-1背包问题的时间复杂度。

用法示例（AcWing0-1背包问题：返回给定容量的背包能够装入物品的最大价值）：

二维dp数组解法

动态规划法“五部曲”：

  1. 确定dp数组及其下标的含义。

    如果使用二维dp数组解决0-1背包问题，则dp[i][j]表示从[0, i]件物品中选取物品，在这些物品能被容量为j的背包容纳的前提下，实现物品价值总和最大。

  2. 确定递推公式。

    对于第i个物品，只存在两种选取方式，选或者不选，并且第i个物品的dp数组的值建立在第i-1个物品的选取结果之上。所以dp[i][j] = max(选，不选) = max(dp[i - 1][j - weight[i]] + value[i], dp[i -1][j])，其中dp[i - 1][j - weight[i]] + value[i]代表选取了第i个物品，需要将背包容量j减去weight[i]，dp[i -1][j]代表不选取，容量不改变，价值也不改变，当然存在dp[i - 1][j - weight[i]] + value[i]的前提是j大于等于weight[i]。

  3. 确定遍历顺序。

    从递推公式可以看出，dp数组的计算依赖于其左侧和上方，所以遍历顺序是从左到右，从上到下。至于是先从左到右，再从上到下（先遍历物品再遍历背包容量），还是先从上到下，再从左到右（先遍历背包容量，再遍历物品），这道题中两种遍历顺序都可以，因为都能够保证递推公式中被依赖的部分先被计算。

  4. 初始化dp数组。

    由递推公式可以看出，dp[i][j]的计算依赖于dp[i - 1][j - weight]和dp[i -1][j]，即位置(i, j)的左上方(i - 1, j - weight[i])和上方(i - 1, j)。所以需要初始化dp数组的最左侧[0, 最大物品下标]和最上侧[0, 最大背包大小]的元素。因为在dp数组的最左侧位置下最大背包的大小为0，所以dp数组的最左侧元素值为0，

一维dp数组（滚动数组）解法

一维dp数组也就是滚动数组，在dp[i]层直接复用dp[i - 1]层。

动态规划法“五部曲”：

  1. 确定dp数组及其下标的含义。

    dp[j]表示容量为j的背包能够装下的物品的最大价值。

  2. 确定递推公式。

    dp[j] = max(dp[j], dp[j - weight[i]] + value[i])，其中dp[j - weight[i]]表示容量为j-weight[i]的背包所装的物品的最大价值。可以看出相对于二维dp数组的写法，这里去掉了dp[i][j]中的i的维度。

  3. 确定遍历顺序。

    只能先遍历物品，再遍历背包：如果是先遍历背包，那么每遍历一个背包容量值，dp数组中最后一个元素是针对于最后一个物品的dp值，在转入到下一个背包容量值时，对应是不是二维dp数组中i-1层的数据，而是最后一个物品所在层的数据，这不符合一维dp数组的作用（在dp[i]层直接复用dp[i - 1]层），即不再是复用第dp[i-1]层，而是复用第dp[n - 1]层（n-1是最后一个物品的下标）。

    背包容量需要从后往前遍历：从后往前遍历的目的是保证每个物品只被添加一次。如果从前往后遍历，以背包容量为4，物品的重量和价值的二维数组为[[1, 15], [3, 20], [4, 30]]为例，在遍历物品0时，如果从前往后遍历，那么dp[1] = dp[1-1] + 15 = 15，dp[2] = dp[2-1] + 15 = 30，可见物品0在dp[2]中被添加了两次，如果是从后往前遍历就不会出现这个问题，dp[2] = dp[2-1] + 15 = 15，dp[1] = dp[1 - 1] + 15 = 15。

  4. 初始化dp数组。

    将0下标的元素初始化为0：因为dp[j]表示容量为j的背包所背物品的最大价值，所以dp[j] = 0。

    将非0下标的元素初始化为0（不大于物品的每个价值）：如果和二维dp数组解法中一样初始化为value[0]，则如果value[0]是非常大的数值，那么dp[i]始终会被更新为value[0]，而其他物品的价值可能无法正确地被纳入计算，这会导致错误的最优解。

  5. 举例推导dp数组。

    以背包容量为4，物品的重量和价值的二维数组为[[1, 15], [3, 20], [4, 30]]为例，dp数组的变化过程为

    用物品0遍历背包：[0, 15, 15, 15, 15]

    用物品1遍历背包：[0, 15, 15, 20, 35]

    用物品2遍历背包：[0, 15, 15, 20, 35]

图的概念

在无向图（Undirected Graphs）中，节点的关系被认为是双向的（bi-directional）。

有向图（Directed Graphs）中， 节点的关系可以指定方向。

未加权图（Unweighted Graphs）的节点和边上均没有权重。

加权图（Weighted Graphs），所加权重可以代表：成本、时间、距离、容量。

邻接矩阵：

​ 图G 是点V 和边E 的集合，记为G=(V, E)。 V表示图中节点的集合，E则表示图中边的集合。

  1. 如果图G是无向图，假如其中有n个结点，我们可以定义一个 二维数组A[n][n]，如果顶点Vi 和顶点Vj 之间存在边Eij， 那么就将A[i][j]设为1，否则设为0。就称二维数组A是图G的邻接矩阵。可见，如果图G是无向图，那么， 如果A[i][j]为1，可以推知A[j][i]也为1。
  2. 如果G是有向图，则如果G的边上有权重，顶点Vi 和顶点Vj 之间存在边Eij，且Eij的权重为Wij，就令A[i][j]为Wij。如果两个顶点Vi 和顶点Vj 之间， 不存在边，那么就记A[i][j]为无穷大。在实际的实现中可以使用整型的最大值代替。

邻接表：对于每个顶点，都维护一个列表，其中包含与该顶点相邻的所有其他顶点。这种表示方法更适用于稀疏图，因为在稀疏图中，每个顶点的相邻顶点数量通常较小，使用邻接表可以节省空间。

连通图：在无向图中，若任意两个顶点vivi与vjvj都有路径相通，则称该无向图为连通图。

强连通图：在有向图中，若任意两个顶点vivi与vjvj 都有路径相通，则称该有向图为强连通图。

生成树：一个连通图的生成树是指一个连通子图， 它含有图中全部n个顶点，但只有足以构成一棵树的 n-1条边。一颗有n个顶点的生成树有且仅有n-1条边 ，如果生成树中再添加一条边，则必定成环。

连通网：在连通图中，若图的每一条边都对应着一个数，称为权；权代表着连接连个顶点的代价，称这种连通图叫做连通网。

最小生成树：在连通网的所有生成树中，所有边的 代价和最小的生成树，称为最小生成树。

并查集

并查集常用来解决连通性问题，也就是判断两个元素是否在同一个集合中的时候（判断两个集合是否连通），或者将两个元素所在的集合合并（通过在两个元素间建立连接来合并两个集合，即使两个集合连通），就可以使用并查集。

并查集主要有三个功能：

  • 寻找节点的根节点。即find(int u)函数，获取节点的根节点。
  • 将两个元素添加到一个集合中。即join(int u, int v)函数，将两个节点连到同一个根节点上。
  • 判断两个元素在不在同一个集合。即isSame(int u, int v )函数，判断两个节点的根节点是不是一个根节点。

并查集的具体操作方法是将除了根节点外的其它所有节点都挂载到根节点下，这样在寻根的时候就很快，只需要一步，这种挂载方式也就是路径压缩的挂载方式。路径压缩的目标是减少树的高度，从而在后续的查找操作中加速。通过在路径上将节点直接连接到根节点，可以使得路径更短，从而提高查找效率。

除了路径压缩，还有一种并查集的挂载方式是按秩合并。秩指的是树的高度，即树中节点层次的最大值。合并思路是将秩小的树合并到秩大的树。

最小生成树

最小生成树就是构造连通网的最小代价的节点连接方法。生成最小生成树有两种常见的算法，Prim算法（普里姆算法）和Kruskal算法（克鲁斯卡尔算法）。

Prim算法

使用Prim算法解决最小生成树问题的步骤如下：

  1. 初始化：
    • 创建一个空的最小生成树集合（开始时只包含初始点）。
    • 创建一个空的距离数组（用于存放顶点及其与最小生成树集合相邻的边的最小权重）。
  2. 添加初始顶点：
    • 将初始顶点加入最小生成树集合。
    • 将初始顶点的所有相邻边加入距离数组。
  3. 主循环：
    • 当最小生成树集合中的顶点数小于图的总顶点数，重复以下步骤：
      • 从距离数组中取出权重最小的边（u, v）（其中u在最小生成树集合中，v不在）。
      • 将顶点v加入最小生成树集合。
      • 遍历顶点v的所有相邻边（v, w），如果顶点w不在最小生成树集合中，则将Math.min(（v, w）的最小权重，距离数组中已有的最小距离)加入距离数组。
  4. 终止条件：
    • 当最小生成树集合中的顶点数达到图的总顶点数时，算法终止。
  5. 输出结果：
    • 最小生成树集合即为最终的最小生成树。

Kruskal算法

使用Kruskal算法解决最小生成树问题的步骤如下：

  1. 初始化：

    • 将图中的所有边按照权重从小到大进行排序。

  2. 主循环：

    • 从排好序的边列表中，依次取出一条边（u, v）和其对应的权重。

    • 检查边（u, v）的两个端点u和v是否在最小生成树集合中已经连接。

    • 如果u和v不在同一个连通分量中（即加入这条边不会形成环路），则将u和v加入同一个连通分量，并将最小生成树集合中的边数加1。

  3. 终止条件：

    • 当最小生成树集合中的边数等于图的总顶点数减1时，算法终止。

  4. 输出结果：

    • 最小生成树集合即为最终的最小生成树。

Kruskal算法通过不断选择权重最小的边来构建最小生成树，同时避免了形成环路，从而保证最小生成树的正确性。与Prim算法相比，Kruskal算法的思路更加关注于边的选择，适用于稀疏图。

算法的时间复杂度主要取决于边的排序和查找操作，一般情况下为O(E * log(E))，其中E为边数。Kruskal算法不需要像Prim算法那样维护优先队列，因此在某些情况下，它可能在实际应用中更高效。

最短路径

对于没有权值的图，最短路径指的是两个顶之间经过的边数最少的路径。

对于有权值的图，最短路径指的是两个顶点之间经过的边上权值之和最少的路径，可以称路径的第一个顶点是源点，最后一个顶点是终点。

寻找最短路径有两种常见的算法，Dijkstra算法（迪杰斯特拉算法）和Floyd算法（弗洛伊德算法）。

Dijkstra算法

使用Dijkstra算法解决最短路径问题的步骤如下：

Dijkstra算法用于解决单源最短路径问题，即从一个特定的起始节点到图中所有其他节点的最短路径问题。以下是Dijkstra算法的解题步骤：

  1. 初始化：
    • 创建一个集合，用于存放已经找到最短路径的节点。
    • 创建一个数组，用于记录起始节点到其他节点的最短距离。初始时，起始节点的距离为0，其他节点的距离为正无穷。
    • 创建一个优先队列（最小堆），用于按距离从小到大访问节点。
  2. 将起始节点加入队列：
    • 将起始节点加入优先队列，其距离为0。
  3. 主循环：
    • 在优先队列不为空的情况下，进行以下步骤：
      • 从优先队列中取出当前距离最小的节点，记为当前节点。
      • 如果当前节点已经被访问过（已在集合中），则跳过。
      • 否则，将当前节点标记为已访问，将其加入集合。
      • 遍历当前节点的所有邻居节点，计算从起始节点经过当前节点到邻居节点的距离。如果这个距离小于邻居节点已知的最短距离，则更新邻居节点的最短距离，并将邻居节点加入优先队列。
  4. 结束：
    • 当优先队列为空时，表示所有节点已经处理完毕，算法结束。
  5. 获取最短路径：
    • 最终，每个节点的最短距离都会被计算出来。你可以通过查询数组中的值，找到从起始节点到其他节点的最短距离。
    • 如果需要还原最短路径本身，你可以借助记录前驱节点的数组，从目标节点逆序追溯回起始节点，得到完整的最短路径。

Floyd算法

使用Floyd算法解决最短路径问题的步骤如下：

  1. 初始化距离矩阵：
    • 创建一个距离矩阵dist，其中dist[i][j]表示节点i到节点j的最短距离。
    • 如果存在一条直接边从节点i到节点j，将dist[i][j]设置为这条边的权重，否则设置为一个较大的值（代表不可达）。
    • 对角线上的元素dist[i][i]应该设置为0。
  2. 核心循环：
    • 使用三重循环遍历所有节点对(i, j, k)：
      • 对于每一对节点(i, j)，尝试通过节点k来检查是否存在更短的路径：dist[i][k] + dist[k][j] < dist[i][j]。如果存在更短路径，更新dist[i][j]为更小的值。
  3. 结束：
    • 当核心循环结束后，矩阵dist中记录了任意两个节点之间的最短距离。
  4. 路径还原（可选）：
    • 如果需要还原最短路径本身，可以借助一个额外的前驱矩阵next，其中next[i][j]表示从节点i到节点j的最短路径上，节点j的前一个节点是什么。
    • 在核心循环中，如果找到了更短的路径dist[i][k] + dist[k][j] < dist[i][j]，则将next[i][j]更新为节点k，表示从节点i到节点j的最短路径需要经过节点k。

排序

如果相等的2个元素，在排序前后的相对位置保持不变，那么这是稳定的排序算法。

执行流程：

  • 冒泡排序

    从头开始比较每一对相邻元素，如果第1个比第2个大，就交换它们的位置。

  • 选择排序

    从序列中找出最大的那个元素，然后与最末尾的元素交换位置

  • 插入排序

    1. 在执行过程中，插入排序会将序列分为2部分，头部是已经排好序的，尾部是待排序的。
    2. 从头开始扫描每一个元素。每当扫描到一个元素，就将它插入到头部合适的位置，使得头部数据依然保持有序。

  • 堆排序

    1. 对序列进行原地建堆（heapify）
    2. 重复执行以下操作，直到堆的元素数量为 1
      1. 交换堆顶元素与堆尾元素
      2. 堆的元素数量减 1
      3. 对 0 位置进行 1 次 siftDown 操作

  • 归并排序

    1. 不断地将当前序列平均分割成2个子序列，直到不能再分割（序列中只剩1个元素）。
    2. 不断地将2个子序列合并成一个有序序列，直到最终只剩下1个有序序列。

  • 快速排序

    1. 从序列中选择一个轴点元素（pivot），假设每次选择 0 位置的元素为轴点元素
    2. 利用 pivot 将序列分割成 2 个子序列
      1. 将小于 pivot 的元素放在pivot前面（左侧）
      2. 将大于 pivot 的元素放在pivot后面（右侧）
      3. 等于pivot的元素放哪边都可以
    3. 对子序列进行a、b操作
      1. 直到不能再分割（子序列中只剩下1个元素）

  • 计数排序

    1. 遍历序列，统计每个数字出现的次数，将出现的次数存储到一个数组中，所以数组的长度需要大于等于最大的数字。
    2. 从数组的最左侧遍历每个数字出现的次数，出现多少次就往结果集中将这个数字添加多少次。

  • 基数排序

    依次对个位数、十位数、百位数、千位数、万位数…进行排序（从低位到高位），适合用于非负整数排序。

  • 桶排序

    1. 创建一定数量的桶（比如用数组、链表作为桶）。
    2. 按照一定的规则（不同类型的数据，规则不同），将序列中的元素均匀分配到对应的桶。
    3. 分别对每个桶进行单独排序。
    4. 将所有非空桶的元素合并成有序序列。

  • 希尔排序

    希尔排序也被称为递减增量排序（Diminishing Increment Sort）。

    1. 设增量序列个数为k，则进行k轮排序。
    2. 每一轮排序中，按照某个增量将数据分割成较小的若干组。
    3. 对一组内部进行插入排序。
    4. 各组排序完毕后，减小增量，进行下一轮的内部排序。
